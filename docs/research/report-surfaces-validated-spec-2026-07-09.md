# Report surfaces — validated improvement spec (2026-07-09)

Three-stage review of every user-facing generated artifact (PDF/CSV/print/email):
1. **Branding/consistency audit** (2 code-exploration agents) — every artifact's header/footer/branding/filename vs what exists.
2. **Content/functionality audit** (3 field-extraction agents) — exact fields rendered vs data held, per artifact.
3. **External validation** (8 web-research agents) — every proposed change confirmed/refuted against AU industry practice: AS 4000/SOPA certification, TMR MRTS50 (full text), TfNSW Q6, VicRoads 160, NATSPEC GEN 009, ISO 9001 §8.7/10.2, NATA rules, ISO/IEC 17025, Xero import spec, construction-claims legal guidance, competitor teardown (Payapps/CONQA/Procore/HammerTech).

Product lane validated: **Payapps = the money schedule; CONQA = the conformance evidence. CIVOS is a better CONQA, not a worse Payapps.** Every dollar CIVOS computes is a chance to disagree with Xero — pass money through, never derive it.

## Findings that override the original proposal (research won)

- **No CIVOS-computed dollars on evidence documents.** Per-lot budget $ and $ contract-position on the claim cover: REFUTED. Show the *physical* position (previous/this/cumulative **%**, lots complete vs total); dollar figures only as labelled pass-through from the claim.
- **No cost impact on NCR PDFs** — industry deliberately keeps money off quality records.
- **No lot allocations on the subbie-facing docket PDF** — internal costing; approver can't verify it.
- **Scheduled claims digest: defer** until a warm HC customer asks (valuable, but new product surface).
- **Test method taxonomy: one optional free-text field only** — the lab report states the method authoritatively.
- **Conformance report must NOT reprint NCRs** — disposition + closure status + reference only; NCR stays the system of record. Nor replace NATA certs — result-vs-limit summarises on top of the attached cert.

## Bug-grade discoveries (fix regardless of batch)

1. **Claim package Declaration falsely asserts** "percentage complete are shown in the lot sections" — never rendered. A signed document asserting content it doesn't contain.
2. **Xero blank `TaxType` = silent GST under-billing.** A present-but-blank TaxType cell imports the line as *untaxed* (does NOT inherit the account default). Default to `GST on Income` (exact org rate-name match required; keep override; `GST Free Income`/`BAS Excluded` for tax-free, never blank).
3. **Xero `DueDate = InvoiceDate`** — taken literally on import → every claim ages as overdue day one. Use the SOPA-derived `calculatePaymentDueDate` (already exists for claims CSV) or invoice date + configurable terms. Also: emit ISO `YYYY-MM-DD` (locale-safe), make AccountCode per-company configurable, help text "import as Tax Exclusive".
4. **Hold-point package section toggles are inert** (`_options` never read).
5. **Company logo silently missing on 4 of 8 PDFs** (diary/docket/test/dashboard callers drop company data) + logo delivery is a flaky client-side fetch (copy HP's server-embedded data-URL pattern).
6. **Docket PDF references subcontractor ABN that's never passed.**

### Batch B — money documents
- **Claim evidence package**: fix Declaration mismatch; per-lot **% previous/this/cumulative** (physical, no $); cover physical cumulative position + labelled pass-through claim total; **itemised evidence per lot** (ITP items who/when/verifier, tests w/ lab + request nos., HP releases w/ signatory) — "a QS cannot certify a count" (AS 4000 reasonable-evidence + SOPA s.13 itemisation); activity-type subtotals (already computed server-side); statutory-declaration pointer on cover (NSW head contractors — mandatory accompaniment).

### Batch C — quality documents
- **Conformance report** (MRTS50 10.1.1(b), 8.2(e), 8.4): test results **vs spec min/max + pass/fail** (top priority); **verifier identity + date per ITP item** (independence: "at least one remove"); completed-at dates; spec clause + acceptance criterion per item; NCR rows → disposition + closure status + reference (open NCR flags lot); roles/titles in certification blocks; use the declared-but-unused authority spec prefixes.
- **Test certificate → retitle "Material Conformance Record"** (labs issue *reports*; "certificate" = implied product certification risk per NATA rules); print lab report identity (lab, report/serial no., sample, date) — the audit anchor; ITP-item link; mandatory footer ("contractor conformance record… not a laboratory report"); optional test-method field (needs migration — flag at build).
- **NCR PDF** (ISO 9001 8.7.2(c)(d); MRTS50/Q6 written-approval rule): **concession block when disposition = concession** (justification + client approval reference + approver + date) — must-fix; verifier identity + date; spec reference (clause + drawing/rev); explicit disposition type (check existing enum first); linked failed-test reference (conditional).
- **HP package**: releaser **role/organisation prominent** (the load-bearing authority fact); lab report refs + spec limits in test table; attachment filenames + type/description (not counts).
- **Hardening check**: public release links — per-recipient single-use tokens + captured email/IP/timestamp on release (Electronic Transactions Act identity-binding). Verify current implementation; gap = small hardening item.

### Batch D — site records & registers
- **Diary PDF** (print-what-you-store): **contemporaneity block** (submitted-at, same-day/late flag, locked status, named submitter — highest legal value, Walter Lilly line); delays w/ lot/location + affected activity; docket provenance per personnel/plant row; deliveries; safety events (occurrence + reference level of detail, not raw narrative); visitors.
- **Delay register CSV**: + lot/location.
- **Registers**: lot CSV + conformedAt/ITP template (+claim link, framed as reconciliation not audit); NCR CSV + raised-by/root cause/closure/status/location; test CSV + verifier + lab report no. (label contractor-verifier vs lab signatory).
- **Reports hub**: surface already-computed fields (claims tab: certification/collection rates, payment refs; test tab: spec limits, lab report nos.; diary tab: row detail, plant by-company).

## Multi-state validation (closed 2026-07-09)

Conformance-report changes (result-vs-limits, verifier identity, acceptance criteria, NCR
closure status) are **confirmed valid for all five build states**: QLD (MRTS50, full text),
NSW (TfNSW Q6), VIC (VicRoads 160), **SA (DIT PC-QA1/PC-QA2 — same model, unit is "Work
Lot")**, **WA (MRWA Spec 201 — ITP acceptance criteria, NATA-endorsed reports, Conformance
Report + NCR)**. No SA/WA-specific mandatory extra fields. Note: verifier-identity field
must be able to capture a Principal/Superintendent hold-point release, not just internal
sign-off.

SOPA itemisation of work: required in **every** state — itemised evidence is nationally valid.

**Supporting statement (subbies-paid declaration) — state-conditional, 3 branches off the
project `state` field:**
1. **NSW / QLD / VIC / ACT** → reminder that a head contractor's claim must be accompanied
   by a supporting statement in the approved form (NSW SOPA s.13(7)-(9); QLD BIF Act; VIC
   s.14(2A); ACT modelled on NSW — moderate confidence, confirm section before relying).
2. **SA** → softer note: no statutory requirement, but Government/DIT contracts require a
   Subcontractor Payment Statutory Declaration.
3. **WA / TAS / NT** → no note.

**Wording guard:** the evidence package must state it is ancillary — it is NOT the statutory
payment claim and NOT the supporting statement. Never imply it satisfies either.
